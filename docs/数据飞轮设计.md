# 数据飞轮设计

数据飞轮负责把系统运行中的反馈变成后续优化数据。

```text
审核轨迹
  -> 人工反馈
  -> hard case 挖掘
  -> synthetic data 生成
  -> SFT / DPO 数据构建
  -> reward scorer 评分
```

## 数据文件

- `audit_logs.jsonl`：每次审核的执行轨迹。
- `feedback.jsonl`：人工反馈。
- `hard_cases.jsonl`：错误样本池。
- `synthetic_cases.jsonl`：合成样本。
- `sft_data.jsonl`：监督微调格式数据。
- `dpo_pairs.jsonl`：偏好优化格式数据。

这些文件默认被 `.gitignore` 排除，避免把本地运行数据提交到 GitHub。
